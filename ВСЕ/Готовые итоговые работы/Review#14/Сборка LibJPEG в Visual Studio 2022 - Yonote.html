    Сборка LibJPEG в Visual Studio 2022

    Первым шагом создаём отдельную директорию для этого проекта, например, LibJPEG.

    Сборка библиотеки

    1. Внутри созданного каталога LibJPEG создаём директорию build-lib, в которой будем собирать библиотеку через саму Visual Studio.

    2. Распаковываем этот архив внутрь build-lib

      zip

      jpegsr9e.zip

      1.16 MB

      Должна получиться следующая структура

      D:\PROJECTS_CPP\LIBJPEG
└───build-lib
    └───jpeg-9e

    3. Внутрь каталога jpeg-9e кладем файл win32.mak.

      mak

      win32.mak

      20.04 kB

    4. Открываем Visual Studio без кода


    5. С помощью PowerShell переходим в директорию LibJPEG/build-lib/jpeg-9e, которая была распакована из архива jpegsr9e.zip.



    6. Прописываем команду NMAKE /f makefile.vc setup-v17


      После этого в директории LibJPEG/build-lib/jpeg-9e должен появиться файл jpeg.sln


    7. Открываем jpeg.sln и устанавливаем конфигурацию x64.


    8. Жмем правой кнопкой мыши на Решение “jpeg” и нажимаем Собрать решение.


      Решение должно собраться


      После этого в каталоге LibJPEG\build-lib\jpeg-9e\Release\x64 должен появиться файл jpeg.lib. Это наша статическая библиотека, которую мы будем использовать в конверторе изображений.


    9. Создаем директорию LibJPEG\sln, рядом с LibJPEG\build-lib


    10. Внутри LibJPEG\sln создаем каталог непосредственно для библиотеки jpeg


    11. Внутри него создаем директории include и lib


    12. В include кладем эти файлы

      h

      1.75 kB
      h

      jerror.h

      14.89 kB
      h

      jmorecfg.h

      15.37 kB
      h

      50.59 kB

    13. В lib создаём 2 каталога Debug и Realese


    14. В оба этих каталога кладем собранную на 8 пункте статическую библиотеку jpeg.lib, которая располагается в D:\Projects_Cpp\LibJPEG\build-lib\jpeg-9e\Release\x64

    Сборка проекта и подключение к нему LibJPEG

    1. Скачиваем архив из теории

      zip

      reading_dog.zip

      965.20 kB

    2. Распаковываем его в LibJPEG\sln


    3. В LibJPEG\sln\ImgLib\CMakeLists.txt сразу указываем абсолютный путь до нашей библиотеки, чтобы не указывать его через консоль или внутри самой Visual Studio

      cmake_minimum_required(VERSION 3.11)

project(ImgLib CXX)
set(CMAKE_CXX_STANDARD 17)

# добавляем новую переменную - путь к LibJPEG
set(LIBJPEG_DIR "D:/Projects_Cpp/LibJPEG/sln/libjpeg") # здесь указывается ВАШ путь

if(NOT LIBJPEG_DIR)
    message(FATAL_ERROR "Please, specify LibJPEG directory via -DLIBJPEG_DIR=<dir>")
endif()
message(STATUS "LibJPEG dir is ${LIBJPEG_DIR}, change via -DLIBJPEG_DIR=<dir>")

set(IMGLIB_MAIN_FILES img_lib.h img_lib.cpp)

# к файлам форматов добавим JPEG
set(IMGLIB_FORMAT_FILES 
    ppm_image.h ppm_image.cpp 
    jpeg_image.h jpeg_image.cpp)

add_library(ImgLib STATIC ${IMGLIB_MAIN_FILES} 
            ${IMGLIB_FORMAT_FILES})

# Include-директории теперь включают LibJPEG
target_include_directories(ImgLib PUBLIC "${LIBJPEG_DIR}/include")

# Флаг INTERFACE обозначает видимость параметра - только для зависимых целей.
# Для самой ImgLib не нужно указывать зависимости компоновки, поскольку
# статическая библиотека не компонуется, а просто архивируется
target_link_directories(ImgLib INTERFACE
    "${LIBJPEG_DIR}/lib/$<IF:$<CONFIG:Debug>,Debug,Release>"
    )

# В качестве зависимости указано jpeg. Компоновщик будет искать файл libjpeg.a
target_link_libraries(ImgLib INTERFACE jpeg)

    4. В файле LibJPEG\sln\ImgLib\jpeg_image.cpp исправляем 52 строку, дописывая L перед "rb" в условии

      if ((infile = _wfopen(file.wstring().c_str(), L"rb")) == NULL)

    5. В LibJPEG\sln\ImgConverter создаем директорию build и переходим в неё


    6. Открываем терминал, желательно не cmd и пишем

      cmake ../ -G "Visual Studio 17 2022"


      В результате появляется проект Negate.sln, с которым мы будем взаимодействовать


    7. Открываем Negate.sln


    8. Нажимаем правой кнопкой мыши на Решение “Negate” и нажимаем Собрать решение.


      Решение без проблем собирается


    9. Нажимаем правой кнопкой мыши по проекту imgconv и назначаем его в качестве запускаемого проекта


    10. Пробуем запустить проект


      Всё завелось.


      Остается лишь указать агрументы main через параметры проекта и положить нужные картинки рядом с исполняемым файлом imgconv.exe.

    11. Кладем входные картинки в директорию LibJPEG\sln\ImgConverter\build


      Чтобы была возможность запуска через консоль, кладем их в директорию LibJPEG\sln\ImgConverter\build\Debug


    12. Указываем аргументы командной строки (аргументы для int main) через свойства проекта

      1. Проект → Свойства imgconv


      2. Отладка → Аргументы команды. Указываем названия входного и выходного файлов


        Обязательно нажимаем “Применить”.

    13. Запускаем проект в режиме отладки


    14. Радуемся :)
